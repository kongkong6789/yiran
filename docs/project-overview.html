1. 项目有什么用

一句话：让运营同学用自然语言提问、协作开会、调用技能脚本， 系统自动检索知识 / 图谱 / 企微文档，必要时经闸机审批后执行业务动作。

对话与协作

  • 对话 Agent：多轮问答、附件、Skill 调用
  • 团队协作 / 小策 bot：房间会话、进度可视化
  • 圆桌 / Council：多角色讨论与方案压缩

知识与经营

  • 知识库检索、本体图谱、经营回路
  • Wiki / RAG / DuckDB 指标与异常
  • MCP：企微文档、NAS 文件注入上下文

执行与治理

  • SOP 编排 → 闸机校验 → Connector 执行
  • Skill 仓库：上传 zip、沙箱跑 scripts
  • 审计、账号、对象（Agents）管理

近期新增（1C/2A）

  • agentctx：跨会话记忆 + 上下文组装
  • Skill 沙箱：超时 / cwd / env / 命令白名单
  • 管理页 /agent-memory

4. 核心链路（当前实现）

用户消息 → Context 组装（记忆 / 会话摘要 / RAG / 图谱 / MCP / NAS / Skill 输出 / 附件） → LLM 生成回复 → Memory 轻量更新（偏好/事实启发式 + 滚动摘要） Skill 脚本 → 物化 workspace → 白名单校验（仅 python + scripts/*.py） → 裁剪 env + 强制 cwd + 超时 → stdout 回填进 Context 业务动作 → Orchestration 编排 → Harness 闸机（允许 / 拒绝 / 需审批） → Connectors 执行 + 审计

7. 怎么优化（工程与体验）

可靠性

数据库与路由

统一 PostgreSQL 多库别名（如 knowledge）配置，避免 USE_POSTGRESQL 切换后别名丢失；迁移与 keepdb 测试流水线固定。

异步任务

小策回复、Skill 长任务、记忆压缩改为 Celery/RQ + 可取消，避免线程内跑 LLM 难观测。

可观测性

为 run_chat 打 span：召回耗时、沙箱耗时、token、记忆是否注入；对接日志与告警。

体验

协作切换

继续强化按房间缓存草稿/附件；大房间消息虚拟列表（已有 virtuoso 可复用）。

引用透明

回复侧展示 refs（知识块、Skill、MCP、记忆条目），用户可一键「忘记这条记忆」。

模型策略

按意图分流：检索问答用快模型，表格/长文用强模型；视觉模型候选已有，可配置化。

安全

  • 密钥：Skill 沙箱已裁剪 env；继续审计 MCP/Connector 日志是否落密钥。
  • 闸机：高风险写操作强制审批流，与 harness 审批单 UI 打通。
  • 提示注入：对上传附件 / Skill stdout 做不可信标签，限制其覆盖系统规则。
