# 良策 AI 对话页双主题改版设计

日期：2026-07-16  
状态：已确认，待实施

## 1. 背景

现有 `/agent` 页面已经具备历史对话、模型选择、Skill、连接器、附件、管理员只读、Markdown 报告等完整能力，但视觉呈现仍偏传统后台：历史栏、主面板、快捷提示和输入区使用相近的边框与卡片重量，信息层级不够鲜明，对话内容不是视觉主角。

本次改版参考 [VoltAgent awesome-design-md](https://github.com/VoltAgent/awesome-design-md) 中 Claude、Intercom 与 Linear 的设计规则，但不复制其品牌。目标是在保留“良策”藏青与金色资产的前提下，为 AI 对话页提供两种可切换的工作氛围：

- 温润智囊：Claude / Intercom 式暖色、安静、长文友好。
- 沉浸指挥：Linear 式深色、精密、适合高专注决策场景。

## 2. 已确认决策

1. 主题切换只影响 `/agent` AI 对话页，不影响顶部全站导航和其他业务页面。
2. 首次进入默认使用“温润智囊”。
3. 用户可通过对话页右上角的太阳 / 月亮按钮即时切换主题。
4. 浏览器记忆用户选择，后续进入时恢复上次主题。
5. 两套主题共用同一页面结构、业务逻辑、中文文案和交互能力。
6. 不使用两套独立页面，也不切换全局 Ant Design 主题。

## 3. 目标与非目标

### 目标

- 让对话内容与输入区成为视觉主角，降低后台表单感。
- 形成清晰的历史栏、对话区、消息、引用与输入工具层级。
- 在温润与深色主题之间提供快速、稳定、可记忆的切换。
- 完整保留现有对话能力，避免视觉改版造成行为回归。
- 在桌面和移动端均保持可用、可读和可操作。

### 非目标

- 不改造顶部全站导航或其他业务页面的主题。
- 不新增后端接口、用户主题字段或跨设备同步。
- 不修改消息协议、会话 API、模型 API、附件 API 或权限逻辑。
- 不在本次范围内重构全站品牌系统。
- 不为温润与深色主题维护两套 React 组件树。

## 4. 视觉设计

### 4.1 共同布局

页面继续采用“历史栏 + 对话工作区”双栏结构，但弱化外层卡片框架：

- 历史栏使用独立底色和细分隔线，不再通过重阴影与主内容竞争。
- 对话页头包含 Agent 身份、连接状态、新对话按钮和主题切换器。
- 消息区使用更宽的阅读列，助手回答优先采用正文排版；报告、表格和代码仍由 `ChatMarkdown` 展示。
- 输入框悬浮于消息区底部，模型、Skill、连接器、附件与发送操作收纳在同一工具条。
- 空状态使用 Agent 标记、简短欢迎语和两到三个任务建议，避免大面积空白。

### 4.2 温润智囊主题

- 画布：暖米白，而非纯白或冷灰。
- 侧栏：略深的暖砂色，形成轻柔分区。
- 主要文本：深炭色；次要文本：暖灰。
- 强调色：克制的陶土橙，用于 Agent 标记、选中状态和发送按钮。
- 卡片：细暖灰边框、低对比阴影、12–16px 圆角。
- 用户消息：淡陶土色底；助手长文尽量不使用厚重气泡。
- 氛围：友好、低压、适合咨询与长文阅读。

### 4.3 沉浸指挥主题

- 画布：深藏青黑，不使用纯黑。
- 侧栏与主区采用两级深色表面，通过发光极弱的金色强调建立层级。
- 主要文本：冷白；次要文本：蓝灰。
- 强调色：良策金，用于 Agent 标记、选中状态和发送按钮。
- 卡片：发丝级蓝灰边框，主要通过表面层级区分，不依赖重阴影。
- 用户消息：深蓝高一级表面；助手长文保持高对比和舒适行距。
- 氛围：沉浸、精密、适合数据复盘与经营决策。

### 4.4 主题切换器

- 位于对话页头右侧，放在“新对话”按钮旁边。
- 采用双按钮分段控件：太阳代表温润，月亮代表沉浸。
- 当前主题必须具备明显的背景、图标颜色和焦点状态。
- 提供 Tooltip、`aria-label` 和键盘焦点样式。
- 主题过渡只作用于颜色、背景、边框与阴影，时长为 200ms；不对尺寸与位置做动画。
- 遵循 `prefers-reduced-motion`，用户要求减少动画时禁用过渡。

## 5. 组件与状态设计

### 5.1 主题状态

在 `AgentChat` 内新增局部主题类型：

```ts
type ChatTheme = "warm" | "command";
```

主题状态只属于对话页。页面根节点通过 `data-chat-theme="warm|command"` 暴露当前主题，CSS 使用该属性切换作用域内变量。

### 5.2 本地持久化

- 使用固定的 `localStorage` key：`liangce_chat_theme`。
- 初始化时读取已保存值，仅接受 `warm` 或 `command`。
- 没有有效值、读取失败或存储不可用时使用 `warm`。
- 切换成功后尝试写入；写入失败不弹错误提示，也不阻断当前页面切换。
- 本次不将主题同步到后端或其他设备。

### 5.3 CSS 作用域

主题变量定义在 `.agent-chat-shell` 和 `.agent-chat-shell[data-chat-theme="command"]` 下，例如：

- 画布、侧栏、主表面、次级表面。
- 主要、次要、弱化文本。
- 边框、强调色、强调色文本。
- 选中态、工具按钮、阴影。

现有全局 `--lc-*` 变量继续服务其他页面。对话页组件应优先消费局部 `--chat-*` 变量，避免深色样式泄漏到顶部导航、Ant Design 全局组件或其他路由。

### 5.4 现有能力保留

以下行为不做改变：

- 历史会话加载、选择与删除。
- 管理员用户筛选和只读预览。
- 新对话与自动保存。
- 模型列表、模型类型标签与本地模型选择。
- Skill 与连接器选择。
- 文件、图片附件与预览。
- Enter 发送、Shift+Enter 换行。
- Markdown、表格、代码、Mermaid、报告块与引用展示。
- LLM、知识、MCP 和 Skill 执行状态标签。

## 6. 数据流与异常处理

主题切换不参与后端数据流：

1. 页面挂载时安全读取本地主题。
2. React 状态确定根节点的 `data-chat-theme`。
3. 用户点击切换器后立即更新 React 状态。
4. CSS 局部变量触发视觉变化。
5. 页面尝试将新主题写入本地存储。

异常处理原则：

- 本地存储异常静默降级，不出现 Toast，不影响聊天。
- 无效主题值忽略并回退温润主题。
- 现有 API 请求失败逻辑保持不变。
- 深色主题下，Ant Design 下拉浮层若由 Portal 渲染到页面外，不强行全局深色化；优先保证作用域与全站安全。对话页内已存在专用 `popupClassName` 的控件可做局部适配。

## 7. 响应式与可访问性

- 桌面端保留历史栏与主工作区。
- 小于现有断点时继续隐藏历史栏，主题切换器始终可见。
- 输入工具条在窄屏允许收缩和换行，不遮挡发送按钮。
- 切换按钮和主要操作保持至少 36px 的可点击尺寸；移动端优先达到 40px。
- 两套主题的正文、弱化文本、边框和按钮状态满足可读对比度。
- 所有交互提供可见键盘焦点，不仅依赖颜色表示选中。
- 支持 `prefers-reduced-motion`。

## 8. 验收与测试

### 自动验证

- `npm run build` 通过，TypeScript 无新增错误。
- 对主题解析与默认值做可单独验证的纯逻辑处理；若项目不引入测试框架，至少通过构建与浏览器行为验证覆盖。

### 浏览器视觉与交互验证

每个主题至少检查：

- 空状态。
- 有普通消息状态。
- 有 Markdown 标题、列表、表格、代码块和报告卡片的状态。
- 有附件与图片预览的状态。
- 加载中状态。
- 管理员只读状态。
- 模型、Skill 和连接器弹层。
- 桌面宽度与移动宽度。
- 太阳 / 月亮鼠标点击与键盘操作。
- 刷新后恢复上次主题；无存储值时默认温润。
- 顶部导航和其他路由不受主题切换影响。

## 9. 完成标准

满足以下条件即视为完成：

1. `/agent` 默认显示温润智囊主题。
2. 用户可通过页头按钮即时切换到沉浸指挥主题并切回。
3. 刷新页面后恢复最后一次选择。
4. 主题样式不影响顶部导航和其他路由。
5. 现有全部聊天能力继续工作。
6. 两套主题在桌面和移动端均通过视觉与交互检查。
7. 前端生产构建通过。
