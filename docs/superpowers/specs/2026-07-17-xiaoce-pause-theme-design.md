# 小策bot暂停生成与局部黑白主题设计

## 背景

当前正式的 AI 对话入口已经迁入 `/collab`，以通讯录中的“小策bot”单聊形式工作。旧的暂停生成实现仍停留在 `codex/chat-pause-skill-packaging` 分支，并且只服务于旧 `/agent` 页面；已经上线到 `main` 的苹果黑白主题同样只作用于旧页面。

本次采用最小迁移：保留协作页面和小策bot现有消息、WebSocket、轮询及知识问答链路，不整体合并旧分支，只把真实取消能力和已确认的黑白主题迁到小策bot。

## 目标

1. 小策bot生成回答时，右下角发送按钮切换为方形暂停按钮。
2. 点击暂停会真实终止本轮后端执行，保留用户消息，不保存未完成回答，并且只显示一次“已暂停本次生成”。
3. 小策bot中间对话区支持纯白与纯黑主题切换，默认白色并记住用户选择。
4. 左侧通讯录、右侧监控、其他协作会话和小策bot Logo/头像颜色保持不变。
5. 尽量沿用同事现有代码结构，不重写协作聊天。

## 不在本次范围

- 不给普通同事私聊或群聊增加主题切换。
- 不改变 `/collab` 的会话结构、实时同步协议或风险监控逻辑。
- 不整体合并旧分支的“会话打包 Skill”功能。
- 不保留部分生成内容，也不提供从中断位置继续生成。
- 不改变 `/agent` 路由和页面。
- 不负责内网服务器部署；代码审核通过后再按现有上线流程更新服务器。

## 方案选择

采用持久化运行记录与协作式取消。

仅使用浏览器 `AbortController` 虽然改动少，但后台仍会继续调用 LLM、Skill 和 MCP，不满足真实暂停。把旧 `/agent` 页面嵌入协作页会破坏小策bot现有消息模型和实时同步，也不采用。

## 后端设计

### 小策bot运行记录

在 `collab` 应用新增独立运行模型，记录：

- 前端生成的 UUID；
- 房间、发起用户和触发消息；
- `running`、`cancelled`、`completed`、`failed` 状态；
- 唯一的暂停占位消息；
- 取消、完成、创建和更新时间及错误信息。

运行记录属于协作会话，不复用旧 `ChatRun` 对 `ChatSession` 的强关联，避免把两套会话模型硬绑在一起。

### 发送与执行流程

1. 小策bot单聊发送消息时，前端生成 `run_id` 并随现有消息请求提交。
2. 后端保存用户消息和运行记录后，继续用现有后台线程执行小策bot回答；发送接口仍然快速返回。
3. 后台线程把统一 `cancel_check` 传入现有 `run_chat`，并在 RAG、图谱、MCP、Skill、图片及 LLM 阶段边界检查取消状态。
4. LLM 使用旧暂停实现已经验证的流式读取方式，在数据块之间检查取消并关闭连接；Skill 子进程和 MCP 调用沿用旧分支的协作式取消方式。
5. 回答生成后，在事务中锁定运行记录：只有仍为 `running` 才创建小策bot消息并标记 `completed`；若已经取消则丢弃结果。
6. 运行状态变化继续通过现有 WebSocket 推送，并由轮询兜底。房间详情返回当前用户在该小策bot房间里的活动运行，刷新或切回页面后仍能恢复暂停按钮。

### 取消接口

新增小策bot运行取消接口，要求：

- 只能取消当前登录用户在当前房间发起的运行；
- `running` 转为 `cancelled`，并以事务创建唯一的“已暂停本次生成”小策bot消息；
- 重复取消返回相同结果，不重复创建消息；
- 已 `completed` 的运行返回冲突，避免把已完成回答伪装成暂停；
- 不存在或不属于当前用户的运行统一返回未找到；
- 取消结果和占位消息通过现有实时通道推送。

## 前端设计

### 运行交互

- 只有当前房间是小策bot单聊且存在活动运行时，发送按钮才变为方形暂停按钮。
- `running` 显示“暂停生成”；点击后进入 `cancelling`，显示“正在暂停”并阻止重复点击。
- 取消成功后，由服务端实时消息显示“已暂停本次生成”，按钮恢复为发送。
- 取消失败时恢复可点击暂停按钮并保留 `run_id`，允许重试。
- 普通协作会话继续使用现有发送按钮和发送逻辑。

### 局部主题

- 复用现有 `agentChatTheme.ts` 的 `light`/`dark` 类型、默认值和本地存储键，确保用户之前选择的黑色主题仍然有效。
- 主题开关只在小策bot会话头部显示。
- `data-chat-theme` 只挂在中间对话区容器，不挂到整个 `/collab` 或 `body`，避免影响左侧通讯录、右侧监控及弹窗。
- 白色主题使用纯白画布、黑色文字和黑色主按钮；黑色主题使用纯黑画布、白色文字和白色主按钮。不新增中性灰主题。
- 消息气泡、分隔线、输入框、滚动条和悬停态使用局部 CSS 变量；小策bot头像、Logo 和用户头像不套用黑白滤镜或强制背景色。

## 一致性与竞态

- 同一用户在同一小策bot房间只允许一个活动运行；生成中再次发送会被前端阻止，后端也做冲突校验。
- 完成和取消通过数据库行锁决定唯一最终状态，保证不会同时出现正式回答和暂停占位。
- 切换房间不会自动取消；返回小策bot房间时从服务端恢复活动运行状态。
- 组件卸载或浏览器断开不会伪造暂停，后台运行仍由持久化状态管理。

## 测试与精简验证

后端聚焦测试：

- 运行创建、所有权、活动运行冲突和状态转换；
- 幂等取消、暂停消息唯一、完成后不可取消；
- 取消后不保存正式回答，以及完成/取消竞态；
- `run_chat`、LLM、MCP 和 Skill 在关键阶段响应取消；
- 小策bot发送接口返回运行状态，房间详情可恢复活动运行。

前端聚焦测试：

- 仅小策bot生成时显示暂停按钮；
- 取消中防止重复点击，失败后允许重试；
- 收到完成或暂停状态后恢复发送按钮；
- 主题默认白色、切换黑色并持久化；
- 主题选择器和局部样式不会出现在其他协作会话。

验证流程保持精简：先跑相关后端测试和前端单测，再各跑一次前端构建与 Django 系统检查，最后只在浏览器验证小策bot的发送、暂停、刷新恢复和黑白切换。

## 验收标准

- 小策bot思考期间可以真实暂停，暂停后不再出现本轮正式回答。
- 用户消息保留，暂停占位只出现一次，随后可以继续提问。
- 黑白主题只改变小策bot中间对话区，刷新后仍保持选择。
- 小策bot Logo/头像颜色、左侧通讯录、右侧监控和其他会话没有视觉变化。
- 现有协作实时消息、轮询兜底、附件发送和风险监控测试无回归。
