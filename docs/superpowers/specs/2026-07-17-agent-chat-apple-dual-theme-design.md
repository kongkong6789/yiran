# 良策对话页 Apple 极简黑白双主题设计

日期：2026-07-17  
状态：视觉方案已确认，待实施

## 目标

只改造 `/agent` 对话界面，使其采用 Apple 式安静、平面、阅读优先的浅色/深色双主题。首次进入使用白色，用户切换后在当前浏览器记住偏好。聊天能力、接口协议、权限和其他路由不变。

本设计替代 2026-07-16 的“温润智囊 / 沉浸指挥”方案。

## 视觉基准

- 已确认预览：`.superpowers/brainstorm/69347-1784253162/content/awesome-design-reference-preview-v3.html`
- 设计规范：[VoltAgent awesome-design-md / Apple DESIGN.md](https://github.com/VoltAgent/awesome-design-md/blob/main/design-md/apple/DESIGN.md)
- 对话结构：助手回答采用无气泡正文，用户消息使用克制的浅层气泡。

## 已确认设计

1. 主题只作用于 `/agent`；全站顶部导航和其他页面保持现状。
2. 主题值为 `light` 和 `dark`，默认 `light`。
3. 使用 `localStorage` key `liangce_chat_theme` 保存当前浏览器偏好；无效值或存储异常时回退 `light`。
4. Logo 保持原有品牌颜色；用户上传图片、图表和报告内容保留原始颜色。
5. 界面使用 Apple 黑、白和系统灰阶，不使用原有深蓝、金色、装饰渐变或彩色强调。
6. 页面采用全屏平面结构：左侧历史栏 + 右侧阅读工作区，不使用外层悬浮大卡片。
7. 浅色主题使用白色主画布与 `#f5f5f7` 历史栏；深色主题使用黑色主画布与近黑分层表面。
8. 正文以 17px、约 1.47 行高呈现；助手回答最大阅读宽度约 690–760px，标题使用 SF Pro Display / 系统字体和紧凑字距。
9. 主要层级依靠留白、字体、整面底色与少量发丝分隔线；UI 卡片、按钮和输入区不使用投影。
10. 输入区固定在对话工作区底部，18px 圆角、轻边框，模型、Skill、连接器、附件和发送能力保持可用。
11. 页头只保留 Agent 状态、模型、必要操作和浅色/深色切换器。
12. 颜色切换为 200ms；遵循 `prefers-reduced-motion`。

## 实现边界

- 使用单一 React 组件树和 `/agent` 局部 CSS 变量，不切换全局 Ant Design theme。
- 不复制浅色/深色两套页面，不重构聊天 API，不新增后端主题字段。
- 现有会话历史、管理员只读、模型、Skill、连接器、附件、Markdown、Mermaid、发送和错误处理逻辑全部保留。
- 对由 Portal 渲染的对话页弹层使用专用 class 做局部主题适配，避免影响其他路由。

## Git 与数据库接入

1. 主题完成并本地提交后，再同步 GitHub `origin/main` 的同事更新；优先使用 rebase，冲突时同时保留主题与同事业务改动。
2. PostgreSQL 凭据只写入已被忽略的 `backend/.env`，不进入源码、设计文档或 Git 历史。
3. 沿用现有架构：SQLite 继续承载 Django 登录、权限和业务模型；PostgreSQL 作为现有数据湖/业务数据建模连接，不迁移默认数据库。
4. 同步和连接完成后停在本地可测试状态，未经用户审核不推送。
