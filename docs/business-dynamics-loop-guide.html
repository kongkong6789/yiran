Part A 问题与概念层

01我们缺什么

已有 Raw、Snapshot、认证指标、FeedbackLoop、Stock/Flow、仿真—— 缺的是可重复规程：从 Excel / ERP 到「这是一个系统」之间，第几步问什么、何时才允许写方程。

原则：先问题与参考模式 → 再因果结构 → 再绑数据 → 再方程与情景。 没有闭环与延迟叙事，只是指标看板。

Part B 建模方法（书中五步）

03书中标准建模五步

Sterman 表 3.1。建模是反复循环；目的是「刀子」——决定该割舍什么。

步骤要回答产物
1. 问题与边界问题？变量？时界？参考模式？问题卡、草图
2. 动态假说内部哪些反馈造成行为？CLD / 存量流量图
3. 写方程决策规则、参数、初值？方程表、量纲一致
4. 测试再现参考模式？极端？灵敏度？对照与灵敏度笔记
5. 政策评估改什么规则？如果—则？干预对比、落地建议

走例：护发精油 70ml · 覆盖天数来回抖

只切这一问题，不建「整个店铺」。

①

明确问题与边界

  • 问题：近 90 天覆盖天数在 8～28 天震荡；猛补后仍短暂断货。
  • 变量：可用 / 在途 / 日均销 / 覆盖天数 / 下单量（暂不含广告、退款）。
  • 时界：回看 90 天，前瞻 30～60 天。
  • 参考模式：锯齿——偏低 → 大补 → 滞后回升 → 再偏低。
产物 → 问题卡 + 参考模式（振荡）
②

动态假说

不怪「供应商慢」，从系统内部讲：

可用↓ → 覆盖↓ → 补货↑ → 下单↑ →（铅期 7～10 天）→ 到货↑ → 覆盖↑ → 补货↓
  • B 调节；振荡 = 寻的 + 延迟。
  • 下单不计在途 → 铅期内重复加单 → 振幅更大。
产物 → 假说一句话 + CLD（type=B）
③

写方程

覆盖天数 = 可用 / 日均销
目标覆盖 = 14 天
缺口 = max(0, 目标×日均 − 可用 − 在途)   // 含在途
到货 = 延迟(下单, 铅期=7)
可用 += 到货 − 日销

初值例：可用 300、在途 0、日均 20 → 覆盖 ≈ 15 天。

产物 → 方程 + 参数（lead_time=7, cover_days=14）
④

测试

  • 仿真是否也出现「补后滞后再跌」？
  • 可用=0 时销出必须停；铅期=0 时振荡应减弱。
  • 改参前先写预期，再跑（避免乱点参数）。
产物 → 对照笔记 · L1→冲 L2
⑤

政策设计

政策模型改动预期
下单扣在途缺口含在途少重复加单
按周补 / 上限减猛补更稳
缩短铅期7→4振荡减弱
仅抬目标覆盖14→21少断货但占资金；未必治根
推荐 → 含在途 + 适度目标；写入 Knowledge SOP
刻意割舍：全店 GMV、多仓、广告闭环、退款。若测试发现销量本身大抖，下一轮再把日销改内生。

05成熟度门禁 & 检查单

L0
问题卡 + 参考模式 + CLD 闭环（可无方程）
L1
Stock/Flow/Delay + 启发式参数 + 只读情景能跑
L2
历史校准；轨迹同相位/量级；有灵敏度笔记
L3
留出样本回测；证据升级 supported/validated
L4
受控预测；高风险仍走闸机审批
未完成 S0–S4 禁止 L1+；未绑同一 Snapshot 认证指标禁止仿真；未做极端/灵敏度禁止 L2+。

每场检查单

  • 一句话说清问题与时界？
  • 画出参考模式形状？
  • Snapshot live 且对账完整？
  • 变量标明存量/流量/单位？
  • 至少一个闭合 R/B + 延迟？
  • 入/出流分开、存量非负？
  • 初值来自同一 Snapshot？
  • 改参前写下预期？
  • 干预改结构而非只加剂量？
  • 业务参与过假说讨论？
